    Prometheus node_exporter网络指标

    2021-03-09
    Tech
    Prometheus, node_exporter

    node_exporter中的网络相关指标由node_network前缀开头。

    我们可以通过访问node_exporter的服务端口，直接查看该节点的各项指标：

    其中，常用指标有

    • node_network_receive_bytes_total：网卡上传累计字节数
    • node_network_transmit_bytes_total：网卡下载累计字节数

    其中选项device代表每个网卡。

    通过Prometheus计算上传速率/下载速率的查询语句：

    • 上传速率：irate(node_network_transmit_bytes_total{device!=“lo”}[1m]) by (instance, job)

    • 下载速率：irate(node_network_receive_bytes_total{device!=“lo”}[1m]) by (instance, job)

    device : 代表网卡

    irate : 分组函数，取的是在指定时间范围内的最近两个数据点来算速率

    [1m] : 分组的时长

    by(instance,job) : 代表按job、instance分组

    grafana免登录

    2020-12-10
    Tech
    Prometheus, grafana

    在ubuntu下，grafana以systemctl管理的service的方式运行，修改默认环境变量配置文件：

    /etc/default/grafana-server文件

    增加以下环境变量配置：

    GF_AUTH_PROXY_ENABLED=true
GF_AUTH_ANONYMOUS_ENABLED=true

    安装部署prometheus

    2020-12-07
    Tech
    Prometheus

    安装步骤 #

    一、安装prometheus #

    部署路径：/usr/local/prometheus

    服务文件：/usr/lib/systemd/system/prometheus.service

    部署过程:

    1. 解压文件，复制到指定路径 # 
    tar zxvf prometheus-2.22.2.linux-amd64.tar.gz
mv prometheus-2.22.2.linux-amd64 prometheus
cp -r prometheus /usr/local/
    2. 创建service服务文件 # 
    vim /usr/lib/systemd/system/prometheus.service

[Unit]
Description=Prometheus
After=network.target
 
[Service]
ExecStart=/usr/local/prometheus/prometheus --config.file=/usr/local/prometheus/prometheus.yml --storage.tsdb.path=/data/prometheus --storage.tsdb.retention.time=180d --storage.tsdb.retention.size=29TB
User=root

[Install]
WantedBy=multi-user.target

    其中，执行命令涉及的参数如下：

    • config.file：配置文件路径。
    • storage.tsdb.path：tsdb存储路径，默认是相对于执行路径的/data目录。
    • storage.tsdb.retention.time：数据保存的最长时间，这里是180天。
    • storage.tsdb.retention.size：数据保存的最大量值，这里最大保存29TB的数据，再有更多的数据，则会删除最早的历史数据来腾出空间。

    加入开机启动：

    systemctl enable prometheus
systemctl start prometheus
